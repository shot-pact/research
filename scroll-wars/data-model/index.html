Scroll Core

The source of truth. Every number in the app derives from this one table. Yesterday's ranks (for delta arrows) are computed live at small scale; cached in Redis at global scale — never persisted in Postgres.

▸daily_scroll_stats HOT
The one write your backend cares about. Upsert per user per local day. Everything projects from here. 5 columns, composite PK — tight enough to stay in buffer cache.

Write Once. Project Everywhere.

ONE UPSERT TRIGGERS THE ENTIRE SIDE-EFFECT CHAIN
⟶ POST /scroll/sync
daily_scroll_stats
App pushes instagram_seconds once per local day. Backend upserts, computes reels_scrolled, then fans out side-effects atomically in a transaction.
⟶
🔥
users (mutate)
Bump lifetime_reels, update streak fields by comparing local_date to last_scroll_date. Single UPDATE, same txn.
🏆
user_achievements (upsert)
O(1) check against users.lifetime_reels + users.current_streak — no aggregate scan needed
⚡
redis: lb:friends:* (invalidate)
Bust leaderboard caches for all accepted friends
📡
ws: score:updated (broadcast)
Fan out to every connected friend socket for live LB · the only real-time channel (no push)
